Overview

Event-driven Java GUI that simulates a small e-store: search items, add to cart, compute totals, and generate invoices with transaction logs.

Tech

  • Java (OOP, Swing/AWT events)
  • Event-driven programming
  • File I/O (CSV read/write)
  • Input validation & errors

Key Files

  • NiledotCom.java
  • inventory.csv (input)
  • transactions.csv (output)

Tasks Supported

  • Search by item ID / name
  • Add / remove from cart
  • Compute subtotal, tax, discounts
  • Finalize order & log invoice

Description

This project implemented an event-driven e-store interface in Java, modeled as “Nile Dot Com.” The application connects GUI actions (button clicks, text input) to backend logic that manages a shopping cart and order lifecycle. Users can search inventory, add or remove items, compute totals (including tax and discounts), and check out. Completed purchases are written to an output log with unique transaction identifiers based on timestamp.

The program reads product data from inventory.csv, validates user input (e.g., item ID existence and stock levels), and handles common error cases through clear messages. On checkout, the app generates a human-readable invoice and appends a structured record to transactions.csv, supporting basic audit and reconciliation.

Skills Learned

I applied event-driven programming by wiring GUI controls to handlers that update application state and the cart view in response to user input. I practiced object-oriented design to keep inventory, cart items, and totals well-structured and testable. Implementing file I/O taught me to parse CSV inputs safely and persist finalized transactions. Finally, I improved error handling by validating item IDs, quantities, and stock, then surfacing informative messages so users can correct issues quickly.

How It Works

  1. Load items from inventory.csv into memory at launch.
  2. User searches by ID or name → matching item details appear.
  3. User selects quantity → “Add to Cart” updates cart and running totals.
  4. “Delete Last Item” backs out the most recent addition if needed.
  5. “Checkout” computes subtotal, tax, discounts, and final total, then writes a row to transactions.csv.
